# Wire Search Web UI — Design Spec

## Overview

Make the Wire subtitle search accessible to non-technical users via a self-contained single HTML file that can be opened in any browser with no installation, no server, and no CLI required.

## Files

- `build.py` — reads `index.json`, injects it into an HTML template, writes `wire-search.html`
- `wire-search.html` — the generated output; ~8.5MB; distribute this file to users

## Build

```bash
python build.py
```

Reads `index.json` from the same directory, embeds its contents as a JavaScript constant in the HTML template, and writes `wire-search.html`. The build script is the only thing that needs to be re-run if subtitles change.

## UI — Minimalist Style

- Black or near-black text on white background, system font stack
- No external dependencies (no CDN, no fonts, no frameworks)
- Minimal spacing, no decorative elements
- Single column layout, max-width ~720px, centered

### Layout (top to bottom)

1. **Header** — "The Wire" as small muted label, "Subtitle Search" as the page title
2. **Search row** — text input (full width) + "Search" button inline
3. **Filter row** — Season dropdown, Episode dropdown, Context (0–5 select), Case-sensitive checkbox — all on one line, compact
4. **Results header** — e.g. `42 results` or `Showing 50 of 312 results` — plain text, small
5. **Result cards** — one per match, no borders/shadows, separated by a thin rule or whitespace:
   - Episode label: `S01E03 — The Buys  [00:38:53 → 00:38:57]` in small monospace
   - Context lines before (if any): muted gray text
   - Matched line: normal weight, query term(s) wrapped in `<mark>` (yellow highlight)
   - Context lines after (if any): muted gray text

### Interactions

- Search triggers on **Enter** keypress in the input or clicking the Search button
- Season dropdown resets Episode dropdown to "All" when changed
- Default result limit: 50 (not exposed in UI — matches search.py default)
- If results are truncated, show a note: `Showing first 50 results — refine your query to see fewer`

## Search Logic (JavaScript)

Mirrors `search.py` exactly:

1. Compile query as `RegExp` with `i` flag unless case-sensitive is checked; fall back to `RegExp(escaped_query)` if the pattern is invalid
2. Filter `INDEX` array by season and/or episode if dropdowns are not "All"
3. Iterate filtered entries; collect matches with context (context entries must share the same season+episode)
4. Highlight: replace matched portion in result text using the compiled regex, wrap in `<mark>`
5. Truncate to 50 results; show truncation note if applicable

## What Is Not Included

- `--rebuild` flag (regenerating `index.json` from SRT files) — CLI only, not needed in the UI
- Regex mode toggle — regex is always on, same as `search.py`; invalid regex silently falls back to literal

## Non-Goals

- No server-side component
- No fuzzy matching
- No pagination (truncation at 50 is sufficient)
- No mobile-specific optimizations (basic responsiveness via max-width is enough)
